如何将代码提交到远程仓库

情景一

在已经clone了仓库的情况下，只需要在当前文件夹下进行仓库修改
步骤一：git add .
提交全部文件修改到缓存区
步骤二：git commit -m <message>
提交代码到本地仓库，并写提交注释
步骤三：git pull
这个命令可以确保本地仓库与远程仓库保持同步。如果有有冲突的地方，合并完再进行步骤四
步骤四：git push
上传本地仓库到远程仓库

情景二

没有关联远程仓库，但是已经完成了部分代码的情况
步骤一：git init
步骤二：git remote add origin <repositories-url>

步骤三：git add .
步骤四：git commit -m <msg>
步骤五：git push -u origin master

情景三

如果远程仓库完全为空，且没有的分支的时候会推送不上去。这个时候需要先创建一个分支git checkout -b master
具体步骤：步骤一：git init
步骤二：git remote add origin <repositories-url>
步骤三：git checkout -b master<branch_name>
步骤四：git add .
步骤五：git commit -m <msg>
步骤六：git push -u origin master<branch_name>

在执行完步骤二后，可以使用git remote -v的命令来查看是否正确关联
在执行完git checkout -b master后，可以通过git branch -r来查看远程分支是否创建成功。
如果远程分支列表中显示了你刚刚创建的分支，则说明远程分支创建成功。

情景四

当拉取当前分支后，有其他开发者又创建了新的分支。
你想要拉去那个新分支，此时，使用git checkout <new_branch_name>
会发现无法拉取。此时需要做的操作是使用git fetch获取远程仓库的更新。
再次git checkout <new_branch_name>可以发现成功切换

注意点

1.在使用开发分支的开发过程中，需要在一定时间合并一下main分支即主分支的代码，以免由于开发进度不一致导致开发分支与线上分支差距过大

2.切记不能合并beta分支代码，beta分支的功能可能与线上差距很大，会引起代码变乱。
如果需要修正某一个错误，可以先在开发分支修改，beta分支合并开发分支后本地运行调试。
